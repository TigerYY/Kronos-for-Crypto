1. Kronos 在单一时间框上的预测原理

模型视角：把 K 线当作「语言」来建模
  • 输入数据：某一时间框（如 1h）的 OHLCV 序列，最近 400 根 K 线（LOOKBACK = 400）。
  • Tokenizer：KronosTokenizer 将价格、成交量进行 归一化 + 分桶，转成离散 token。
  • Transformer：Kronos 对 token 序列做 自回归预测，生成未来 12 根 K 线（PRED_LEN = 12）。
  • 反归一化：KronosPredictor 将预测 token 还原为未来的 open/high/low/close/volume。
原始 OHLCV (400 根) --Tokenizer--> Token 序列 --Kronos Transformer--> 未来 Token --反归一化--> 未来 12 根 OHLCV 序列

在代码中，单一时间框预测对应于 CryptoSimulator.predict(df)： 取 df[['open','high','low','close','volume','amount']]，构造未来 12 个时间戳， 然后调用 predictor.predict(...) 得到未来的 K 线序列。

2. 多时间框架预测整体流程（端到端）

DataFetcher：拉取多时框 K 线
➜
Kronos：对每个时框独立预测
➜
MultiTimeframeStrategy：加权融合涨跌预期
➜
RiskManager：止盈止损 & 仓位控制
➜
VirtualPortfolio：执行买卖 & 记录

步骤 1：多时框数据获取 trading/data_fetcher.py

  • 接口：fetch_multi_timeframe(symbol, ['15m','1h','4h','1d'], LOOKBACK)
  • 加密货币：通过 ccxt binance 获取 OHLCV。
  • 传统期货：如 ES=F，通过 yfinance 获取。
  • 缓存机制：不同时间框设置 TTL（如 15m→60s，1h→120s），降低 API 压力。

步骤 2：逐时框调用 Kronos 预测 crypto_simulator.py

  • 对每个有足够 LOOKBACK 的时框 tf：
  • 调用 sim.predict(df_tf) 得到未来 12 根 K 线。
  • 取预测序列的最后一个 close 作为该时框的目标价。
  • 结果存入字典：predictions[tf] = pred_close_tf。

步骤 3：多时间框加权融合 trading/strategy.py

  • 使用 MultiTimeframeStrategy.generate_signal(predictions, current_price)。
  • 核心思想：将各时框的预测涨跌幅按权重加权平均。
  • 权重示例：15m=0.15, 1h=0.25, 4h=0.30, 1d=0.30。
  • 输出一个 Signal(action, confidence, change_pct, reasons)。

步骤 4：风控与真实下单 risk_manager.py + VirtualPortfolio

  • 优先检查：止损 / 止盈 条件是否触发。
  • 若需买入/卖出，再结合：
  • · 最大持仓比例（默认 80%）
  • · 单次买入不超过总资产的 20%
  • · 最小单笔 10 USDT
  • 通过 VirtualPortfolio 更新余额和持仓，并写入 simulation_log.csv。

3. 多时间框加权与信号判定逻辑

MultiTimeframeStrategy.generate_signal

对于每个时框 tf，假设 Kronos 给出的预测收盘价为 pred_price[tf]，当前现价为 current_price，则计算该时框的预测涨跌幅并加权：

# 对每个有效时框 change_pct_tf = (pred_price[tf] - current_price) / current_price weighted_change += change_pct_tf * weight_tf total_weight += weight_tf # 归一化后得到综合预测涨跌幅 normalized_change = weighted_change / total_weight
  • 方向：normalized_change > 0 → 看涨 → BUY；反之 SELL。
  • 阈值：小于 threshold（默认 0.5%）直接视为 HOLD，避免频繁交易。
  • 强信号阈值：大于 strong_threshold（默认 1.5%）视为高置信度强信号。

若某个时间框数据拉取失败（pred_price is None），会在 reasons 中标记为 "tf=N/A"，并且不参与加权。

信心度 confidence 计算

信心度不仅与涨跌幅大小有关，还考虑了参与投票的时间框数量：

abs_change = |normalized_change| if abs_change < threshold: action = 'HOLD' confidence = 0.0 else: # 基础置信度：在 [threshold, strong_threshold] 之间线性插值 if abs_change >= strong_threshold: base_confidence = 0.85 else: ratio = (abs_change - threshold) / (strong_threshold - threshold) base_confidence = 0.5 + ratio * 0.35 # 0.5 ~ 0.85 # 多时框加成：valid_count 越多，加成越高 total_tf = len(TIMEFRAME_WEIGHTS) # 目前为 4 data_factor = 0.8 + 0.2 * (valid_count / total_tf) confidence = round(base_confidence * data_factor, 2)
  • 含义：涨跌幅越大、参与的时间框越多，confidence 越接近 1。
  • 避免单时框过弱：即使只有 1 个时间框有效，也通过 data_factor 保证 置信度不会太低，仍有机会触发交易。
  • 理由展示：reasons 字段会记录如 "15m=↑0.80%"、 "4h=↓1.20%" 这样的微解释，前端直接展示。

5. 多时间框设计的直觉与优点

  • 减少单一时间框噪声：5m/15m 等短周期容易被局部波动「骗线」，引入 4h/1d 可用中长期趋势做「锚」。
  • 权重区分周期角色：在 MultiTimeframeStrategy.TIMEFRAME_WEIGHTS 里， 4h / 1d 权重最高，15m 权重最低，符合「长周期定方向，短周期找入场点」的交易直觉。
  • 连续的信号强度而不是二元决策：通过 change_pct 与 confidence， 策略可以区分「略看多」与「极强看多」，为后续仓位调整预留空间。
  • 易于扩展：回测或线上只需调整 TIMEFRAME_WEIGHTS、threshold 等参数， 就可以快速测试不同多时间框组合和风格。
